# ADR Index

## Purpose
1. `docs/adr/` is the canonical, immutable decision log for new process and engineering decisions.
2. Each ADR captures a single decision and is never rewritten after acceptance (only superseded by later ADRs).

## Naming
1. File format: `NNNN-short-title.md`.
2. Start with `0001` and increment by one.

## Required Sections
1. `Title`
2. `Status` (`Proposed`, `Accepted`, `Superseded`)
3. `Date`
4. `Context`
5. `Decision`
6. `Consequences`
7. `Supersedes` (optional)

## Template
Use [0000-template.md](0000-template.md) for all new ADRs.
